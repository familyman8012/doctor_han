# Medihub - 멀티 에이전트(서브 에이전트) 운영 가이드

## 목적

Claude Code의 컨텍스트 윈도우 한계를 전제로, 작업을 역할별로 분리하여 품질을 안정화합니다.

## 권장 모델: “연구원/기획자/검토자/문서작성자”

- `@explorer`: 탐색(읽기 전용)
- `@planner`: 계획(계획 파일만 쓰기)
- `@reviewer`: 리뷰(읽기 전용)
- `@spec-writer`: 스펙(PRD/TSD만 쓰기)

> 실제 코드 구현은 기본 에이전트가 전담합니다. (세션 분산으로 생기는 컨텍스트 단절을 피하기 위함)

## 언제 누구를 부를까

- “어디에 있지?” → `@explorer`
- “어떻게 구현할까?” → `/plan-feature` + `@planner`
- “스펙이 비어있다/충돌한다” → `@spec-writer`
- “머지 전에 봐줘” → `/code-review` + `@reviewer`

## 운영 규칙(드리프트 방지)

- 계획(plan)은 SSOT가 아님. PRD/TSD에 없는 요구/설계를 plan에 추가하지 않는다.
- DB/RLS/API/Auth 같은 고위험 변경은 `/workflow`의 승인 포인트에서 반드시 요약 후 진행한다.
