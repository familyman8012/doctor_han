# Execute: 계획 실행

## 계획 파일: $ARGUMENTS

## 미션

`.agents/plans/` 디렉토리의 구현 계획을 정확히 따라 기능을 구현합니다.

**핵심 원칙**: 계획에 명시된 순서대로 작업을 수행하고, 각 단계마다 검증합니다.

## 실행 프로세스

### 1. 계획 로드 및 검증

1. 계획 파일 읽기: `.agents/plans/$ARGUMENTS.md`
2. 컨텍스트 참조 파일들 모두 읽기
3. 참조 문서 확인
4. 따라야 할 패턴 이해

### 2. 순차적 작업 실행

계획의 **단계별 작업** 섹션을 위에서 아래로 실행:

각 작업에 대해:
1. 작업 내용 확인
2. 참조 패턴 확인
3. 구현 수행
4. **즉시 검증 명령어 실행**
5. 실패 시 수정 후 재검증

### 3. 작업 유형별 처리

**CREATE** (새 파일 생성):
- 지정된 경로에 파일 생성
- 패턴 참조 파일을 미러링
- 프로젝트 컨벤션 준수

**UPDATE** (기존 파일 수정):
- 파일 먼저 읽기
- 최소한의 변경으로 목표 달성
- 관련 없는 코드 수정 금지

**ADD** (기능 추가):
- 기존 코드 구조 유지
- 올바른 위치에 삽입
- import 문 정리

### 4. 단계별 검증

각 Phase 완료 후:
```bash
cd app
pnpm type-check  # 타입 에러 확인
pnpm lint        # 린트 에러 확인
```

### 5. 최종 검증

모든 작업 완료 후 **검증 명령어** 섹션의 모든 명령 실행:

```bash
cd app

# Level 1: 문법 & 스타일
pnpm lint
pnpm type-check

# Level 2: 테스트 (프레임워크 설정 후 사용)
# pnpm test

# Level 3: 빌드
pnpm build
```

## 실행 규칙

### 필수 준수 사항

1. **순서 엄수**: 작업을 건너뛰거나 순서 변경 금지
2. **즉시 검증**: 각 작업 후 검증 명령어 실행
3. **패턴 준수**: 참조 파일의 패턴 정확히 따르기
4. **최소 변경**: 계획에 없는 "개선" 금지

### 에러 처리

검증 실패 시:
1. 에러 메시지 분석
2. 해당 작업만 수정
3. 재검증
4. 3회 실패 시 중단하고 보고

### 금지 사항

- 계획에 없는 추가 기능 구현
- 관련 없는 코드 리팩토링
- 주석, docstring 추가 (요청 시 제외)
- 테스트 코드 건너뛰기

## 완료 리포트

실행 완료 후 제공:

### 실행 요약
- 완료된 작업 수
- 각 Phase 완료 여부

### 검증 결과
- 모든 검증 명령어 결과
- 통과/실패 상태

### 생성/수정된 파일
- 파일 경로 목록
- 주요 변경 내용

### 완료 체크리스트
- [ ] 모든 작업 순서대로 완료
- [ ] 모든 검증 통과
- [ ] 테스트 통과 (테스트 프레임워크 설정 시)
- [ ] 린트/타입 에러 없음
- [ ] 빌드 성공

### 다음 단계 (해당 시)

- 배포 관련 참고사항
