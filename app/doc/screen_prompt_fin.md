# Medihub 최종 화면(Screen) 프롬프트 모음 (Fin)

목표: `app/doc/business.md` + `app/doc/test.csv` 기준으로, **“완성된 최종 제품”의 전체 화면 목록(IA)** 을 한 번에 뽑고, 각 화면을 “프롬프트만으로 웹 디자인을 생성”할 수 있도록 화면별 프롬프트를 정리한다.  
주의: 여기서는 MVP/확장 같은 단계 구분 없이 **최종 상태의 페이지를 전부** 나열한다.

---

## 1) 사용법

1) 아래 `BASE PROMPT`를 먼저 붙여넣기  
2) 원하는 화면의 `SCREEN PROMPT`를 이어서 붙여넣기  
3) 도구가 지원하면 결과를 **Figma 프레임** 또는 **React(+Tailwind) 코드**로 출력받기

---

## 2) BASE PROMPT (항상 포함)

```text
당신은 시니어 프로덕트 디자이너이자 UX 라이터입니다.

프로덕트: “메디허브(Medihub)” — 병의원(특히 한의원) 개원/운영을 위한 B2B 업체 검색·비교·문의(리드) 매칭 플랫폼.
컨셉: “의료계의 크몽 + 다나와” (신뢰/표준화/비교/리드 전환 중심)

사용자 역할(Role)
- Guest(비로그인)
- Doctor(한의사)
- Vendor(업체/파트너)
- Admin(운영자)

디자인 톤 & 원칙
- 신뢰감/전문성/깔끔함/빠른 업무 처리 느낌 (과한 광고/과장 금지)
- 의료·B2B SaaS 스타일: 여백 충분, 경계선/그림자 은은, 라운드 10~14px
- 접근성: WCAG AA 대비, 키보드 포커스, 폼 오류/헬퍼 텍스트 명확
- 한국어 UI 카피(레이블/버튼/빈상태/에러)로 작성
- 날짜: YYYY.MM.DD / 금액: ₩ 천단위 구분

레이아웃 규칙
- Desktop: max width ~1200px, 12-column grid
- Mobile: 섹션 스택 + 주요 CTA는 sticky 고려
- Marketplace(탐색) 화면: 상단 헤더(로고/검색/카테고리/유저메뉴)
- Dashboard(Doctor/Vendor/Admin): 좌측 사이드바 + 상단바 + 메인 컨텐츠

공통 컴포넌트 가이드
- Cards, Tables, Filter chips, Tabs, Stepper, File uploader, Modal/Drawer, Toast, Pagination
- Loading skeleton / Empty state / Error state를 반드시 설계

리드(문의) 상태 표준(배지 색상 일관)
- 신청완료, 견적대기, 협의중, 진행중, 계약, 보류, 종료, 취소

출력 형식(가능하면 그대로)
1) Screen title / role / primary goal
2) Layout breakdown (sections + components)
3) UI copy (Korean microcopy 포함)
4) Data requirements (필드/엔티티)
5) Interaction notes (validation, filters, sorting, pagination)
6) States: loading / empty / error / success

코드 출력이 가능하다면
- React + Tailwind CSS 기반의 컴포넌트 구조로 출력
- (가능하면) Next.js App Router 페이지 단위로 생각
```

## 3) 전체 페이지 목록(최종 IA)

### 3-1. 공통/마케팅(Guest 포함)
- 홈/탐색(메인)
- 카테고리 탐색(대/중/소)
- 검색 결과(업체 리스트)
- 업체 상세(Guest/Doctor에서 보이는 버전 차이 포함)
- 업체 비교(다나와식)
- 외부 쇼핑몰/소모품 통합검색
- 헬프센터(FAQ/공지/가이드) 목록 + 아티클 상세
- 이용약관 / 개인정보처리방침
- 404 / 500 등 시스템 에러 페이지

### 3-2. 인증/계정(공통)
- 로그인
- 회원가입: 역할 선택(한의사/업체)
- 한의사 회원가입(서류 포함)
- 업체 회원가입(서류 포함)
- 휴대폰 본인인증(PASS 연동 전/후 UI)
- 소셜 로그인/가입(네이버/카카오/구글) 컴포넌트/에러 상태
- 아이디 찾기
- 비밀번호 재설정
- 인증 상태(검수 대기/승인/반려) + 재제출

### 3-3. Doctor(한의사) 영역
- 온보딩: 관심 카테고리 선택
- 마이페이지 대시보드
- 내 프로필/인증정보 관리(서류 재업로드 포함)
- 알림 센터(인앱) + 읽음 처리
- 알림 설정(채널/유형)
- 찜(즐겨찾기) 업체
- 내 문의함(리드 목록)
- 문의 상세(타임라인/대화/견적/첨부/체크리스트/메모)
- 문의 생성(업체 상세에서 진입)
- 리뷰 작성
- 내 리뷰 목록/관리

### 3-4. Vendor(업체/파트너) 영역(Partner Center)
- 파트너 대시보드(KPI)
- 업체 프로필 관리(기본정보/서비스 소개/가격/지역/카테고리)
- 포트폴리오 관리(등록/수정/삭제)
- 받은 리드함(리드 목록)
- 리드 상세(상태 관리/대화/견적 제출/첨부/체크리스트/메모)
- 리뷰 관리(받은 리뷰/답글/이의제기)
- 광고/우선노출 캠페인 신청/관리
- 정산/결제(크레딧 충전, 결제 내역, 세금계산서)
- 성과 리포트/통계(노출/클릭/리드/ROI/캠페인별)
- 알림 센터(인앱) + 알림 설정

### 3-5. Admin(관리자) 콘솔
- 관리자 로그인
- 관리자 대시보드(핵심 운영 지표)
- 인증 승인/반려 큐(한의사/업체)
- 사용자(한의사) 목록/상세/상태 제어
- 업체 목록/상세/상태 제어
- 카테고리 관리(트리 CRUD)
- 배지/라벨 정책 관리(빠른응답/프라임 등)
- 광고 슬롯/캠페인 운영(노출 정책/승인/기간/예산)
- 과금/정산/요금제 관리(CPC/CPM/CPL/정액) + 리포트/CSV
- 환불/보상 정책 엔진 + 심사 큐
- 리드 모니터링(SLA) + 대화/첨부 열람(관리자 가시성)
- 통합 메시징(카카오 알림톡/이메일) 설정 + 템플릿 + 발송 로그
- 리뷰 모더레이션(신고/블라인드/복구/금칙어)
- 신고 처리/제재 이력(파이프라인)
- 헬프센터 문서 관리(FAQ/공지 CMS)
- SEO/메타/OG 템플릿 관리
- 데이터 Export(회원/업체/리드/정산/캠페인) + 설정 백업/복원
- 감사 로그(Audit Log) 뷰어

### 3-6. 임상/치료 솔루션(케이스 DB + 가이드라인)
- 임상 허브(케이스 DB 메인)
- 케이스 검색/필터/유사도 검색
- 케이스 상세(메타데이터/경과/처방/부작용/첨부)
- 케이스 업로드(템플릿/검증/익명화 안내)
- 내 케이스 관리(개인/학회 단위, 공개 범위)
- 인사이트 리포트(질환/처방 트렌드/효과 분석)
- 한의표준임상진료지침(CPG) 검색/상세
- (옵션) AI 진료 보조(요약/질문 생성/유사 케이스 추천)
- 관리자: 케이스 검수/블라인드, 태그/메타데이터 표준 관리

## 4) SCREEN PROMPTS (최종)

### S-001. 공통: 글로벌 헤더/푸터/내비게이션 패턴

```text
메디허브의 전역 레이아웃(헤더/푸터/내비게이션) 패턴을 디자인해줘.

헤더(마켓플레이스)
- 로고 “메디허브”
- 글로벌 검색바(placeholder: “업체/서비스를 검색하세요”)
- 카테고리 진입(드롭다운 또는 메가메뉴)
- 우측: 로그인/회원가입 또는 유저 메뉴(알림, 마이페이지/파트너센터, 로그아웃)

대시보드(Doctor/Vendor/Admin)
- 좌측 사이드바: 현재 역할에 맞는 메뉴 그룹화
- 상단바: 페이지 타이틀, 브레드크럼, 알림 아이콘, 프로필 메뉴

푸터
- 회사/고객센터/약관/개인정보/공지 링크
- 사업자 정보 영역(placeholder)

반응형 동작(모바일)
- 헤더 축소 + 검색 접근성 유지
- 사이드바는 햄버거 → drawer
```

### S-002. 홈/탐색(메인)

```text
화면: 홈/탐색(메인)
Role: Guest/Doctor/Vendor
Goal: 카테고리 진입 + 검색 시작 + 신뢰 요소로 탐색 전환.

필수 섹션
- 히어로: 가치제안 1줄(“병의원 개원·운영 업체를 빠르고 믿을 수 있게 찾으세요”) + 큰 검색바
- 카테고리 퀵그리드(대분류 8개)
- 추천 업체 / 빠른응답 업체 / 신규 등록 업체 섹션(카드)
- 신뢰 섹션: “실제 의료인 리뷰”, “인증된 업체”, “표준화된 비교”

상태
- 로그인 전/후 헤더 차이
- 추천 데이터 없을 때: 관심 카테고리 선택 유도
```

### S-003. 카테고리 탐색(대/중/소)

```text
화면: 카테고리 탐색
Goal: 대/중/소 카테고리를 빠르게 선택하고 해당 리스트로 이동.

필수
- 3단 구조를 직관적으로 보여주는 UI(메가메뉴/트리/컬럼 3개)
- “전체 보기” + 인기 카테고리 하이라이트
- 선택한 카테고리 breadcrumb(대 > 중 > 소)

모바일
- 바텀시트 단계형 선택(대→중→소)
```

### S-004. 검색 결과(업체 리스트)

```text
화면: 업체 검색 결과(리스트)
Role: Doctor 중심(Guest는 일부 기능 제한 가능)
Goal: 조건(필터/정렬)로 후보를 좁히고 상세/문의로 이동.

필수
- 좌측 필터(데스크탑) / 바텀시트(모바일)
  - 평점, 예산(₩), 빠른응답 토글, 인증 뱃지 토글, 지역(옵션)
- 정렬 드롭다운: 인기순/추천순/최신순/응답순
- 결과 카드: 썸네일, 업체명, 1줄소개, 별점/후기수, 가격대, 배지(빠른응답/인증/AD)
- 스폰서드(AD) 블록(명확한 AD 표기)
- 페이지네이션 또는 무한스크롤 + 결과 수

상태
- 로딩 스켈레톤 / 결과 없음(필터 완화 제안) / 오류
```

### S-005. 업체 상세(Doctor/Guest 겸용)

```text
화면: 업체 상세
Goal: 정보를 충분히 제공하고 “문의하기”로 전환.

상단
- 갤러리(썸네일), 업체명/1줄소개, 배지(빠른응답/인증/세금계산서/AD)
- 별점 요약(평균/후기수) + 공유/찜 + Primary CTA “문의하기”

본문(탭 또는 앵커)
1) 서비스 소개(프로세스 포함)
2) 가격 정보(구간형 표 + 견적형 안내)
3) 포트폴리오(이미지/영상 링크)
4) 리뷰(정렬, 신고, 평점 분포 요약)
5) FAQ

우측 레일(데스크탑)
- 빠른 문의 미니카드(주요 필드 일부 + CTA)
- 응답률/평균응답시간 KPI(있으면)

권한/게이팅
- Guest/미인증 Doctor: “인증 후 문의 가능” 안내 + 인증 이동 CTA
```

### S-006. 업체 비교(다나와식)

```text
화면: 업체 비교(2~4개)
Goal: 표준화된 항목으로 빠르게 비교하고 문의 결정을 돕기.

필수
- 상단 고정(compare header): 업체 카드(썸네일/이름/배지/별점) + 제거 버튼
- 비교 테이블 항목:
  - 가격대/견적형 여부, 제공 지역, 카테고리, 포트폴리오 수
  - 응답률/평균 응답시간
  - AS 정책(placeholder), 세금계산서 가능, 프라임 등 배지
  - 리뷰 수/평점
- “비교에 업체 추가” 검색/선택 UI
- 각 업체별 CTA: “문의하기”
```

### S-007. 외부 쇼핑몰/소모품 통합검색

```text
화면: 소모품/외부 쇼핑몰 통합검색
Goal: 여러 쇼핑몰 결과를 한 화면에서 비교하고 외부로 이동.

필수
- 검색바 + 필터(카테고리/가격/배송)
- 쇼핑몰별 결과 묶음(Shop card) + 내부 상품 리스트 프리뷰
- 상품 row: 이미지/상품명/가격/배송/외부 링크 버튼
- 고지: “외부 쇼핑몰로 이동합니다.”

상태
- 로딩/빈결과/외부 링크 오류 안내
```

### S-008. 헬프센터(목록)

```text
화면: 헬프센터 홈(FAQ/공지/가이드)
Goal: 검색과 카테고리로 빠르게 문서를 찾기.

필수
- 상단 검색바
- 탭: FAQ / 공지 / 이용가이드
- 문서 리스트(태그, 조회수/최종수정일 optional)
- 추천 문서 섹션(최근/인기)
```

### S-009. 헬프센터(아티클 상세)

```text
화면: 헬프센터 아티클 상세
Goal: 내용을 읽고 다음 행동(문의/설정/관련 문서)으로 이동.

필수
- breadcrumb, 제목, 최종수정일
- 본문(목차/앵커), 관련 문서
- “도움이 되었나요?” 피드백(예/아니오) + 추가 의견
```

### S-010. 이용약관 / 개인정보처리방침

```text
화면: 법적 문서 페이지(약관/개인정보)
```

### S-509. AI 진료 보조(옵션) — 케이스 요약/질문 생성

```text
화면: AI 진료 보조(옵션)
Goal: 케이스 텍스트를 요약하고, 추가로 확인할 질문/유사 케이스를 제안.

필수
- 입력: 케이스 요약/메모 붙여넣기 또는 케이스 선택
- 출력 탭:
  1) 요약(구조화)
  2) 추가 질문 리스트(“추가로 확인하면 좋은 항목”)
  3) 유사 케이스 추천
- 고지: “의학적 판단의 최종 책임은 의료인에게 있습니다.”
```
